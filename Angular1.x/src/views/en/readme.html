Angular v1.x Template

The template provides configured and automated work environment with the possibility of further development functionality. It uses technologies such as NodeJS, NPM, Bower, GulpJS and Karma. The basic configuration is prepared for the development of Web application client-side in technology Angular v1.x - MVC Framework.

An additional advantage is a very good cooperation with my other project Restful API - NodeJS & ExpressJS which providing support and more functionality on the server-side about that you can read here.

Usage

Project Angular 1.x Template is available only via Github. Installation examples below. With first two required is version control system Git.

  1. Using console create a new directory and go into him: 
    mkdir project-directory && cd project-directory
    In a new directory initialize environment by using command git init and then call: 
    git pull repositoryhttps://github.com/mateuszarchicinski/Angular1.x-Template
  2. In the console, use a command to clone entire repository, which thereby creates a new directory: 
    git clone repositoryhttps://github.com/mateuszarchicinski/Angular1.x-Template
  3. Manually download the latest stable version from the Releases Page.

After completing any installation. Go to your project and call command npm install and bower install.

At this point, you can start using all defined tasks from gulpfile.js. Table with all tasks here.

Configuration

Global settings are available in file project.config.js, which allows you to change:

  • host (used for SEO),
  • application name,
  • supported languages,
  • value of href attribute in tag base,
  • name of configuration file and data,
  • directory work and production (also requires manual update file .bowerrc),
  • accesses FTP,
  • keys to external APIs,
  • tracking ID Google Analytics,
  • application ID Facebook Developers.

Additionally, it can help us to store all sorts of variables which we will use individually. In file gulpfile.js we refer to it by variable PROJECT_CONFIG.

Gulp tasks

Tasks are run from the console, being at the same time in project directory by typing keyword gulp in combination with name and optional arguments task to do.

gulp name optional arguments

Example command looks like this gulp pug --lang pl (nr: 6).

The following table contains information concerning name, optional arguments, and description for each defined task.

Name Description

sass:css
  1. Uses file main.scss or main.sass.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Initiates creation of Source Map based on main file SASS (gulp-sourcemaps).
  4. Compiles synchronously SASS code to CSS code with output style nested (gulp-sass).
  5. Adds appropriate methods with prefixes to CSS code. Taking into account support for specific browsers with configuration object (gulp-autoprefixer).
  6. Saves Source Map to file main.css.map.
  7. Saves resulting CSS code to file main.css.
  8. Injects new CSS code to overview page (browser-sync).

sass:lint
  1. Uses files with extension .scss and .sass.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Validates SASS code under selected rules
    in configuration object (gulp-sass-lint).
  4. Displays formatted messages
    about detected inconsistencies in SASS code (Formatter).

js:hint
  1. Uses files with extension .js.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Validates JS code and optimisation possibilities (gulp-jshint).
  4. Displays formatted messages
    about detected errors and optimisation possibilities in JS code (Reporter).

js:test
  1. Runs unit tests in environment Karma which is configurable via file karma.conf.js. That file allows choosing test framework and much more things about that you can read here.

jade:pug
  1. Uses files with extension .jade.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Converts file extension .jade to .pug (gulp-rename).
  4. Saves files with a new extension .pug
    in similar locations to files .jade.
  5. Removes not necessary files with extension .jade (del).

pug
--lang pl / en
  1. Checks if passed additional arguments to process task. Sets the initial value of variable langValue to "pl" and overwrites it in a case when otherwise noted.
  2. Uses files with extension .pug.
  3. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  4. Prepares a data object to pass it inside stream to gulp-pug (gulp-data).
  5. Compiles files .pug returning completed HTML code
    with filled information from a data object (gulp-pug).
  6. Injects tags Link and Script with paths to source installed libraries CSS or JS. Between these comments in HTML code (wiredep): 
    <!--bower:css-->
    // <link href="">
    <!--endbower-->

    <!--bower:js-->
    // <script src="">
    <!--endbower-->
  7. Saves resulting HTML code in index-en.html or index-pl.html and view files.

pug:lint
  1. Uses files with extension .pug.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Validates PUG code under selected rules in configuration object (gulp-pug-lint).
  4. Displays formatted messages
    about detected inconsistencies in PUG code.

html
  1. Uses files with extension .html.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Filters HTML code for inserted comments: 
    <!--build:css main.min.css-->
    // <link href="">
    <!--endbuild-->

    <!--build:js main.min.js-->
    // <script src="">
    <!--endbuild-->
    Replacing them with individual tags Link and Script relating to source files main.min.css
    and main.min.js arising as result of combination of all attached libraries, CSS or JS located between that comments (gulp-useref).
  4. Checks type of files being processed in the stream. By calling appropriate minify methods for files
    with extension .css and .js (gulp-clean-css / gulp-uglify).
  5. Saves resulting CSS and JS code to files in production directory.

html:hint
  1. Uses files with extension .html.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Validates HTML code under selected rules in configuration object (gulp-htmlhint).
  4. Displays formatted messages
    about detected inconsistencies in HTML code (Reporter).

html:minify
  1. Uses files with extension .html.
  2. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  3. Minify HTML code according to settings passed in configuration object (gulp-htmlmin).
  4. Saves resulting HTML code in the same source file locations on which task was started (overwriting)

server

  1. Runs locally listening server on work directory at address http://localhost:3000/ (visible in console).

    Providing accessory methods allow you to refresh your browser eg. when you make changes to any file or dynamic injection of CSS styles without reloading the preview page.

    It also provides application preview from any device connected to local area network at external address (External - visible in console).

    More information about configuration and capabilities of Browser Sync module here.

watch

  1. In consequence to files changed it runs appropriate tasks:
    FilesTasks
    .sass or .scssnr: 2 and 1
    .jsnr: 3 and 11
    project.data.json and .pugnr: 7 and 6
    .htmlnr: 9 and 11

clean
  1. Removes production directory with all content. Always runs at the beginning of creating a new application build (del).

copy
  1. Copies so called Static files from work directory to production directory:
    • /files/*
    • /fonts/*
    • /img/*
    • /*.png
    • /*.xml
    • /*.ico

images
--option advanced
  1. Checks if passed additional arguments to process task. Sets file extensions for optimisation, optimisation module and arguments for selected module.
  2. Both standard (without additional arguments) and advanced option (with additional arguments) will use images
    with extension .jpg and .png. In standard option is two more types .svg and .gif.
  3. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  4. Standard images optimisation is done via module gulp-imagemin.
  5. Advanced images optimisation is done via module gulp-tinify which uses TinyPNG API.
  6. Saves optimised images in the same source image locations on which task was started (overwriting).

images:sprite
--name _sprite
  1. Checks if passed additional arguments to process task. Sets the initial value of variable spriteCssName to "_sprite" and overwrites it in a case when otherwise noted.
  2. Uses images with extension .jpg, .png and .gif.
  3. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  4. Merge all images from directory img > sprites_sources in one image composed of many. Returns automatically generated file .scss including completed mixins to use, allows displaying any of these images (gulp-spritesmith).
  5. Checks type of files being processed
    in stream (gulp-if). Saves files
    with extension .png in directory img and .scss in directory sass > components > sprites.

upload
--upload
  1. Checks if all FTP connection variable values are defined in file project.config.js and whether passed additional arguments to process task.
  2. Uses all files from production directory.
  3. Reports about encountered errors in a stream without interrupting execution further pipes (gulp-plumber).
  4. Uploads files to FTP server, passing location through variable DESTINATION (vinyl-ftp).

build
nr: 6, 15, 17

  1. Runs a sequence of gulp tasks in the specified order (run-sequence):nr: 13, 2, 1, 3, 7, 6, 9, 8, 10, 14, 15, 17

    As a result of completing such a complex task process, you will create a new application build ready for further testing or upload to the production environment.

build:server
nr: 6, 15, 17
  1. Invokes gulp task build18 after that, it creates a locally listening server on the newly created production directory.

gulp / default
nr: 6

  1. Runs a sequence of gulp tasks in the specified order (run-sequence):nr: 2, 1, 3, 7, 6, 9, 11, 12

    When above sequence is completed. Creates a work environment that provides many useful features. To improve
    and accelerate a process of creating a new application.

More info and source code to review in file gulpfile.js.

Testing

Runs with the task gulp js:test. To support unit testing, listening changes in source files and reporting errors has been prepared environment Karma. Standard configuration uses the framework Jasmine, simulator PhantomJS and reporter Spec. Files configuration and unit tests you can find Project directory > Test > Karma.conf.js / Spec.
