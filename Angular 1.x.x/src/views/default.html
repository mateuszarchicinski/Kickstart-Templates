Angular v1.x Szablon

Szablon udostępnia skonfigurowane i zautomatyzowane środowisko pracy z możliwością dalszej rozbudowy. Wykorzystuje do tego takie technologie jak NodeJS, NPM, Bower, GulpJS i Karma. W podstawowej konfiguracji jest przygotowany do rozwoju aplikacji webowej po stronie klienta w technologii Angular v1.x - MVC Framework.

Przygotowany szablon aplikacji webowej Angular v1.x udostępnia w pełni skonfigurowane i zautomatyzowane środowisko pracy za pomocą takich technologii jak NodeJS, NPM, Bower, GulpJS i Karma.

Użycie

Projekt Angular 1.x Szablon jest dostępny wyłącznie na Github. Przykłady instalacji poniżej. Do dwóch pierwszych instalacji wymagany jest system kontroli wersji Git.

  1. Za pomocą konsoli utwórz nowy katalog i przejdź do niego: 
    mkdir katalog-projektu && cd katalog-projektu
    W nowym katalogu zainicjuj środowisko komendą git init i następnie wywołaj: 
    git pull repozytoriumhttps://github.com/mateuszarchicinski/Angular1.x-Template
  2. W konsoli wywołaj polecenie sklonowania całego repozytorium, które tym samym utworzy nowy katalog: 
    git clone repozytoriumhttps://github.com/mateuszarchicinski/Angular1.x-Template
  3. Manualnie pobierz najnowszą stabilną wersje z strony publikacji szablonu.

Po wykonaniu dowolnej instalacji. Przejdź do katalogu projektu i wywołaj polecenia npm install i bower install.

W tym momencie można zacząć korzystać z wszystkich zdefiniowanych zadań w gulpfile.js. Tabela zadań dostępna tutaj.

Konfiguracja

Globalne ustawienia są dostępne w pliku project.config.js, który umożliwia zmianę:

  • nazwy aplikacji,
  • nazwy plików konfiguracyjnych i danych,
  • katalogów pracy i produkcji (wymagana również manualna zmiana w pliku .bowerrc),
  • dostępów FTP,
  • podania kluczy do zewnętrznych API.

Dodatkowo może posłużyć nam do przechowywania różnego rodzaju zmiennych z których będziemy korzystać indywidualnie. W pliku gulpfile.js odwołujemy się do niego poprzez zmienną PROJECT_CONFIG.

Gulp zadania

Lista zdefiniowanych zadań w,poniżej. Polecenia wykonujemy z poziomu konsoli przebywając w głównym katalogu projektu. Do każdego zadania dodatkowo krótki opis,z podziałem na zadania pojedyncze i złożone. gulpfile.js

Pojedyncze:

  • gulp sass:css - Tworzy główny plik main.css i main.css.map na podstawie pliku main.scss.
  • gulp sass:lint - Sprawdza kod Sass i raportuje o błędach w plikach .sass /.scss.
  • gulp js:hint - Sprawdza kod JavaScript i raportuje o błędach w plikach .js.
  • gulp js:test - Uruchamia testy jednostkowe, nasłuchuje zmian w plikach źródłowych
    i raportuje o błędach.
  • gulp jade:pug - Konwertuje pliki .jade na .pug.
  • gulp pug - Tworzy pliki .html na podstawie plików .pug (Opcjonalne argumenty: --lang pl / en).
  • gulp pug:lint - Sprawdza kod Pug i raportuje o błędach w plikach .pug.
  • gulp html - Na podstawie wskazanych ścieżek w pliku .html do plików .css i .js łączy
    je w pojedyncze zmniejszone pliki źródłowe.
  • gulp html:hint - Sprawdza kod HTML i raportuje o błędach w plikach .html.
  • gulp html:minify - Po odpowiednim skonfigurowaniu zmniejszy kod HTML w plikach .html.
  • gulp server - Uruchamia serwer nasłuchujący na katalog pracy (src).
  • gulp watch - W następstwie zmian w plikach uruchamia wskazane zadania.
  • gulp clean - Usuwa katalog produkcyjny (dist).
  • gulp copy - Kopiuje wymagane pliki z katalogu pracy (src) do katalogu produkcyjnego (dist).
  • gulp images - Standardowa optymalizacja grafik.
  • gulp images:optimized - Optymalizuje grafiki za pomocą Tinypng API.
  • gulp images:sprite - Utworzy tak zwanego Sprite i plik .scss z grafik znajdujących się w folderze sprites_sources (Opcjonalne argumenty: --name _nazwa).
  • gulp upload - Umożliwia automatyczny upload plików na serwer ftp (Opcjonalny argument: --upload).
  • gulp build:server - Uruchamia serwer nasłuchujący na katalog produkcyjny (dist).

Złożone:

  • gulp build - Wywoła sekwencja poleceń:
clean > sass:lint > sass:css > js:hint > pug:lint > pug > html:hint > html > html:minify > copy > upload
  • gulp build:server - Uruchamia zadanie gulp build po czym tworzy serwer nasłuchujący na nowo powstały katalog produkcyjny.
build > build:server
  • gulp / gulp default - Wywoła sekwencja poleceń:
sass:lint > sass:css > js:hint > pug:lint > pug > html:hint > server > watch

Testowanie

Uruchamia się za pomocą zadania gulp js:test. Do obsługi testów jednostkowych, nasłuchu zamian w plikach źródłowych i raportowania bledów zostało przygotowane środowisko Karma. Standardowa konfiguracja wykorzystuje framework Jasmine, symulator PhantomJS i reporter Spec. Pliki konfiguracji i testów znajdują się Folder projektu > Test > Karma.conf.js / Spec.
