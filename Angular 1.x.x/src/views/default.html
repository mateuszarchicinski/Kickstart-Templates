Gulp zadania

Lista zdefiniowanych zadań w gulpfile.js poniżej. Dodatkowo krótki opis do każdego polecenia z podziałem na dwie grupy pojedynczych i złożonych poleceń.

Pojedyncze:

  • gulp css - Tworzy główny plik main.css i main.css.map na podstawie pliku main.scss.
  • gulp js:hint - Sprawdza kod JavaScript i raportuje o błędach.
  • gulp js:test - Uruchamia testy jednostkowe, nasłuchuje zmian w plikach źródłowych
    i raportuje o błędach.
  • gulp jade:pug - Konwertuje pliki .jade na .pug.
  • gulp pug - Tworzy pliki .html na podstawie plików .pug (Opcjonalne argumenty: --lang pl / en).
  • gulp html - Na podstawie wskazanych ścieżek w pliku .html do plików .css i .js łączy
    je w pojedyncze zmniejszone pliki źródłowe.
  • gulp html:hint - Sprawdza kod HTML i raportuje o błędach.
  • gulp html:minify - Zmniejsza nasz kod HTML.
  • gulp server - Uruchamia serwer nasłuchujący na katalog pracy.
  • gulp watch - W następstwie zmian w plikach uruchamia wskazane zadania.
  • gulp clean - Usuwa katalog produkcyjny.
  • gulp copy - Kopiuje wymagane pliki z katalogu pracy do katalogu produkcyjnego.
  • gulp images - Standardowa optymalizacja grafik.
  • gulp images:optimized - Optymalizuje grafiki za pomocą Tinypng API.
  • gulp images:sprite - Utworzy tzw. Sprite i plik .scss z grafik znajdujących się w folderze sprites_sources (Opcjonalne argumenty: --name _nazwa).
  • gulp upload - Umożliwia automatyczny upload plików na serwer ftp (Opcjonalny argument: --upload).
  • gulp build:server - Uruchamia serwer nasłuchujący na katalog produkcyjny.

Złożone:

  • gulp build - Wywoła sekwencja poleceń:
clean > css > js:hint > pug > html:hint > html > html:minify > copy > upload
  • gulp build:server - Uruchamia zadanie gulp build po czym tworzy serwer nasłuchujący na nowo powstały katalog produkcyjny.
build > build:server
  • gulp / gulp default - Wywoła sekwencja poleceń:
css > js:hint > pug > html:hint > server > watch

Testowanie

Uruchamia się za pomocą zadania gulp js:test. Do obsługi testów jednostkowych, nasłuchu zamian w plikach źródłowych i raportowania bledów zostało przygotowane środowisko Karma. Obecna konfiguracja wykorzystuje framework Jasmine, symulator PhantomJS i reporter Spec. Pliki konfiguracji i testów znajdują się Folder projektu > Test > Karma.conf.js / Spec.
